# Content Analysis

## Topic
养生食疗小程序用户使用场景 - 展示产品如何解决不同人群的养生需求

## Data Type
- 用户故事/场景 (User Scenarios)
- 问题-解决方案结构
- 多用户画像展示

## Complexity
- 中等复杂度
- 3个独立场景，各有明确的问题-解决方案路径
- 需要清晰的空间分隔来区分不同场景

## Tone
- 温暖、关怀
- 生活化、接地气
- 科技赋能传统养生

## Audience
- 中老年用户及其子女
- 关注健康的家庭
- 产品展示/营销用途

## Source Language
中文 (zh)

## User Language
中文 (zh)

## Content Structure
1. 场景A：日常调理（血压问题）- 症状搜索 → 找食谱 → 视频烹饪
2. 场景B：预防保健（关节问题）- 体质/节气推荐 → 发现食谱系列
3. 场景C：子女关爱（睡眠问题）- 远程查找 → 分享给家人

## Design Requirements
- 需要清晰区分3个场景
- 每个场景包含：用户画像、问题、解决方案、产品功能
- 适合产品展示或营销推广
